Common questions

The ones that come up before someone signs up.

How accurate is the detection?
It depends on the input type and threat freshness. For URLs indexed by Google Safe Browsing or reported by our community, accuracy is very high. For novel phishing domains (registered in the last 24 hours), we use domain age, heuristics, and WHOIS signals to catch what blocklists haven't seen yet. We surface per-finding severity so your code can make risk-appropriate decisions rather than relying on a single binary flag.
What's included in the free tier?
The free tier gives you 100 scans per calendar month, 1 API key, and full access to all 9 detection signals — including Google Web Risk. It's a hard cap, so you'll never receive a surprise charge. When you hit 100, the API returns a 429 with a clear message until the next month resets.
How do you handle privacy and GDPR?
Scans are stored against your API key (not against end-user identifiers), so we never receive PII unless you embed it in the address string — which you shouldn't. For phone number lookups, we process the number as a signal but do not store the raw value beyond the scan record. If you need a DPA or have GDPR-specific requirements, contact us before you go to production.
What's your uptime SLA?
Free and Starter plans come with best-effort availability — the API is deployed on Heroku Standard dynos and has been running without issues, but there's no formal SLA. Pro gets informal priority in incident response. Enterprise plans include a contractual uptime SLA with response-time commitments. If your product depends on this API being up, talk to us about Enterprise.
Can I self-host?
Not currently. Self-hosting would require you to provision and maintain API keys for Google Web Risk, WHOIS lookups, and the community database — effectively rebuilding the service. If data residency or air-gapped deployment is a hard requirement, reach out and we can discuss Enterprise options.
How fast is the API?
Median latency sits around 150–180ms. p95 is under 300ms on warm dynos. Cold starts on a free Heroku instance can occasionally add 2–3 seconds after a period of inactivity — if you're on paid plans, dynos stay warm. All 9 analyzers run concurrently, so total latency scales with the slowest signal source, not their sum.
